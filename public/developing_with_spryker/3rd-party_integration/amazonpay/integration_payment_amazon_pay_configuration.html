Amazon Pay - Configuration

To setup the Amazon Pay initial configuration, use the credentials you received after registering as an Amazon seller:

$config[AmazonpayConstants::CLIENT_ID] = 'amzn1.application-oa2-client.someclientid';
 
$config[AmazonpayConstants::CLIENT_SECRET] = 'thisisasecret';
$config[AmazonpayConstants::SELLER_ID] = 'thisisasellerid';
$config[AmazonpayConstants::ACCESS_KEY_ID] = 'thisisanaccessid';
$config[AmazonpayConstants::SECRET_ACCESS_KEY] = 'thisisasecretkey';

Next, specify your country and shop:


$config[AmazonpayConstants::REGION] = 'DE';
$config[AmazonpayConstants::STORE_NAME] = 'The Shop';

For development purposes sandbox mode must be enabled:

$config[AmazonpayConstants::SANDBOX] = true;

The ERROR_REPORT_LEVEL parameter is used for internal purposes and specifies the log verbosity level.

There are three options:

  1. Log all API responses.
  2. Log errors only.
  3. Disable logging.
$config[AmazonpayConstants::ERROR_REPORT_LEVEL] = 'ERRORS_ONLY';


According to Amazon Pay restrictions, a can run either on a localhost domain or on SSL. If it is not possible to use localhost, then SSL should be installed and configured. For testing purposes, register a test account in the dashboard.
